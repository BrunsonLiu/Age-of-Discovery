# 大航海时代交互网站方案

## 1. 项目定位

目标：构建一个“回顾大航海时代”的交互式网站，主视觉为全屏航海地图，结合加勒比海盗主题，让用户直观体验航海探索、贸易、海战和历史事件。

用户价值：
- 让对大航海时代好奇的人感受时代过程
- 给加勒比海盗爱好者提供冒险代入感
- 通过游戏化交互讲述历史背景

---

## 2. 核心体验

### 2.1 视觉结构
- 全屏地图作为主视觉
- 左侧/右侧信息面板：舰队状态、港口详情、事件日志、任务板
- 底部操作栏：时间推进、行动按钮、提示卡片
- 叠加层：风向、洋流、航线、海盗区域

### 2.2 交互方式
- 地图拖拽、缩放
- 点击港口显示详情
- 选择或绘制航线
- 侧边面板操作补给、贸易、修理、接受任务
- 弹窗/卡片显示事件决策结果

### 2.3 玩法要素
- 船队出发准备：船只、补给、船员、物资
- 航线模拟：距离、风向、洋流、风险
- 贸易系统：港口买卖价格差、利润计算
- 事件系统：风暴、海盗、疾病、事故、历史事件
- 任务系统：主线剧情 + 支线探险

---

## 3. 技术架构建议

### 3.1 前端
- 框架：`React` 或 `Vue 3`
- 地图：`Mapbox GL JS` / `Leaflet` + 自定义风格图层
- 样式：`Tailwind CSS` / `Ant Design` / `Element Plus`
- 状态管理：`Redux Toolkit` / `Pinia`
- 过渡与动效：`GSAP` / CSS 动画

### 3.2 数据层
- 静态配置：港口、商品、事件、风向、历史剧本
- 运行时逻辑：前端模拟航海流程
- 存档机制：`localStorage` / `IndexedDB`
- 可选后端：`Node.js` / `NestJS`（若需要多人进度或排行榜）

---

## 4. 核心模块设计

### 4.1 地图层模块
- 全屏地图视图
- 港口标记
- 航线弧线与船只移动动画
- 风向和洋流覆盖
- 隐藏/显示海盗威胁区域

### 4.2 舰队与航程模块
- 船只属性：速度、耐久、装载、补给
- 舰队状态：船员、食物、淡水、金钱、士气
- 航行计算：距离 + 风向 + 洋流 + 事件
- 航程推进：按日/按周更新状态

### 4.3 贸易与港口模块
- 港口商品库存与价格表
- 买入/卖出逻辑
- 港口服务：修理、招募、补给、任务
- 价格波动与利润计算

### 4.4 事件系统模块
- 事件类型：自然、敌对、疾病、任务、历史
- 触发条件：位置、时间、航线、随机概率
- 结果影响：资源变化、船员变化、任务更新
- 决策分支：玩家选择影响结果

### 4.5 任务与剧情模块
- 主线：大航海探索与加勒比海盗故事
- 支线：贸易任务、护送、寻宝、救援
- 叙事形式：卡片、弹窗、旁白式文本
- 历史注释：关键事件附历史小知识

---

## 5. 最小可行产品（MVP）

### 5.1 MVP 内容
- 一个可交互全屏地图
- 3 个主要港口
- 1 支船队可选择航线
- 侧边面板显示基本状态
- 简单事件触发：风暴或海盗遭遇

### 5.2 MVP 价值
- 快速验证地图交互和航线体验
- 让用户立即感受“大航海如何进行”
- 便于后续迭代加入更多剧情与系统
